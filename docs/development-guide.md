# Development Guide: Lost and Found

## Prerequisites

- **Flutter SDK** >= 3.3.0 (Dart SDK >= 3.3.0 <4.0.0)
- **Android Studio** or **VS Code** with Flutter extensions
- **Xcode** (for iOS/macOS builds, macOS only)
- **CocoaPods** (for iOS/macOS, macOS only)
- **Google Maps API Key** (required for map functionality on Android/iOS)
- **Firebase Project** configured (project: `lostfoundapp-1e774`)

## Setup

```bash
cd lost_and_found

# Install Dart dependencies
flutter pub get

# iOS only: Install CocoaPods
cd ios && pod install && cd ..
```

### Google Maps API Key

The app uses `google_maps_flutter` which requires platform-specific API key configuration:

- **Android:** Add API key to `android/app/src/main/AndroidManifest.xml` as a `<meta-data>` entry
- **iOS:** Add API key in `ios/Runner/AppDelegate.swift` via `GMSServices.provideAPIKey()`

**Note:** The current codebase does not have the Google Maps API key configured in the Android manifest. The iOS Pods include GoogleMaps, suggesting it was configured at some point for iOS.

### Firebase Configuration

Firebase is pre-configured via `lib/firebase_options.dart` (generated by FlutterFire CLI).

To reconfigure:
```bash
flutterfire configure
```

## Build & Run

```bash
# Run on connected device or emulator
flutter run

# Run on specific platform
flutter run -d chrome         # Web
flutter run -d ios             # iOS simulator
flutter run -d android         # Android emulator

# Build release APK
flutter build apk

# Build release iOS
flutter build ios

# Build web
flutter build web
```

## Static Analysis

```bash

```

Lint config: `analysis_options.yaml` includes `package:flutter_lints/flutter.yaml`.

## Testing

```bash
# Run all tests
flutter test

# Run a specific test file
flutter test test/widget_test.dart
```

**Current test state:** The only test file (`test/widget_test.dart`) is the **default Flutter template** that tests a counter app. It will **fail** because the app no longer has a counter. There are no meaningful tests.

## Environment Configuration

- No `.env` file is used
- Firebase API keys are hardcoded in `lib/firebase_options.dart` and `android/app/google-services.json`
- No flavor/environment separation (dev/staging/prod)
- No backend URL configuration (all Firebase SDK direct calls)

## CI/CD

No CI/CD pipeline is configured. No `.github/workflows/`, `Fastfile`, or similar files exist.

## Known Development Issues

1. **Widget test will fail** — default template, not updated for this app
2. **No Google Maps API key in Android manifest** — maps may not load on Android
3. **`flutter_map` + `latlong2` declared in pubspec.yaml but unused** — dead dependencies
4. **`firebase_options.dart` not imported in `main.dart`** — `Firebase.initializeApp()` is called without passing `DefaultFirebaseOptions.currentPlatform`, which may cause issues on some platforms
5. **Poppins font referenced but not declared** — will silently use system default
6. **multidex dependency in build.gradle** — `com.android.support:multidex:2.0.1` (AndroidX migration may be needed)
