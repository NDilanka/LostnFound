Lost and Found Campus Mobile Application
## 1. Problem Definition
University campuses are usually large and busy places where students and staff move around
frequently during the day. Every day, people bring many personal belongings with them such as
student ID cards, wallets, keys, books, bags, laptops, and mobile phones. Because students attend
different classes, visit libraries, cafeterias, and other facilities, it is easy for them to accidentally
misplace their belongings.
Losing personal items can cause many problems for students and staff. For example, losing a
student ID card can make it difficult to access certain campus services. Losing a wallet or mobile
phone can also be stressful and inconvenient. Many people spend a lot of time trying to find their
lost belongings, but often they are unsuccessful.
Currently, there is no simple and organized system for reporting lost and found items within
many campuses. When someone loses an item, they usually try to find it by asking friends,

checking classrooms, or posting messages on social media groups. Some students also place
notices on campus notice boards. However, these methods are not always effective because the
message may not reach the person who found the item.
On the other hand, when someone finds an item on campus, they may not know how to return it
to the owner. They might leave it where they found it, keep it temporarily, or give it to campus
security. In some cases, the owner may never know where the item was taken.
Because of these issues, it is clear that a better solution is needed. A centralized system that
allows students and staff to report lost and found items could greatly improve the chances of
returning items to their rightful owners.
This project proposes the development of a Lost and Found Campus Mobile Application. The
main purpose of this application is to create a platform where users can easily report lost items
and found items using their smartphones. The application will allow users to search for items and
contact the person who reported them. By providing this service, the application can help reduce
the number of permanently lost items and make campus life more convenient for everyone.




- Scope of the Project
## Project Objective
The main objective of this project is to design and develop a mobile application that helps
students and staff manage lost and found items within the campus. The application will provide a
simple and user-friendly platform where users can report items they have lost or found.
Another objective of this project is to create a centralized system that organizes all lost and found
information in one place. Instead of relying on multiple platforms such as social media or notice
boards, users will be able to access all relevant information through a single application.
The application will also aim to improve communication between users. If someone finds an
item and posts it in the application, the owner can quickly contact that person and arrange a way
to collect the item.

## Target Users
The application is designed for people within the university campus. The main users of the
system will include:
- University students
- Academic staff members
- Non-academic staff members
- Campus security staff
These users will be able to report lost items, post found items, search for items, and
communicate with other users.
## In Scope
The following features will be included in the development of the application:
- A user registration system that allows users to create an account
- A login system for secure access to the application
- The ability to post details about lost items
- The ability to post details about found items
- A system to browse and search for items
- A page that displays detailed information about each item
- A contact option that allows users to communicate with the person who posted the item
These features will form the main functionality of the application.
Out of Scope
Some advanced features will not be included in this project due to time and complexity
limitations. These include:
- GPS tracking of lost items
- Integration with external social media platforms
- Online payment systems
- Support for multiple universities or campuses
These features may be considered for future improvements of the application.

- Features of the Application

The Lost and Found Campus Mobile Application will include several important features to make
the system useful and easy to use.
## User Registration
Users will be able to create an account in the application by providing basic information such as
their name, email address, and password. This account will allow them to access all the functions
of the application.
Creating user accounts also helps ensure that all item posts are associated with a specific user,
which improves accountability.
## User Login
After registering, users will be able to log in using their email address and password. Once
logged in, they will be able to access the main features of the application such as posting items
and searching for items.
## Reporting Lost Items
If a user loses an item on campus, they will be able to report it through the application. The user
will enter details such as:
- Name of the item
- Description of the item
- Location where it was lost
- Date when it was lost
- Users may also upload an image of the item to make identification easier.
## Reporting Found Items
Users who find an item on campus can also report it through the application. They can provide
details about the item and the location where it was found.
This information will allow the owner of the item to recognize it and contact the person who
found it.
Searching and Browsing Items
The application will display a list of all lost and found items that have been posted by users.
Users can scroll through this list to see the available items.
A search function will also be available to help users quickly find items based on keywords.

## Item Details Page
Each item listed in the application will have its own details page. This page will provide
complete information about the item, including its description, location, date, and image if
available.
Users can review this information to determine whether the item belongs to them.
Contacting the Poster
If a user believes that an item belongs to them, they will be able to contact the person who posted
the item. This feature will help users communicate and arrange how the item can be returned.
## User Profile
The application will include a user profile page where users can manage their personal
information. They will also be able to view the items they have posted in the system.
This feature allows users to keep track of their lost or found item reports.








## 4. Wireframe Diagrams

Wireframes are simple layouts that show how the screens of the application will look before the
actual development begins. Wireframes help designers and developers understand how the
application will be structured and how users will interact with it.
Wireframes for this project can be created using tools such as Figma or Adobe XD. These tools
allow designers to create simple visual representations of the application interface.
The following wireframes will be included in this proposal.
## Login Screen
The login screen will allow users to enter their email address and password to access the
application. New users will also have the option to register for an account if they do not already
have one.
















## Home Screen
The home screen will display a list of lost and found items that have been posted by users. It will
also include a search bar that allows users to quickly find specific items.















## Post Item Screen
This screen will allow users to report a lost or found item. The user will be able to enter details
such as the item name, description, location, and date. An option to upload an image will also be
available.














## Item Details Screen
The item details screen will display complete information about a selected item. This will help
users determine whether the item belongs to them.















## Profile Screen
The profile screen will allow users to view their personal information and manage the items they
have posted in the application.
