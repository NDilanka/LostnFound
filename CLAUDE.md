## Project Overview

**Lost and Found** тАФ a Flutter mobile app for posting and discovering lost/found items. Uses Firebase backend (Auth, Firestore, Storage) with Google Maps for location tagging and image uploads.

## Build & Run Commands

All commands run from the `lost_and_found/` directory:

```bash
flutter pub get          # Install dependencies
flutter run              # Run on connected device/emulator
flutter build apk        # Build Android APK
flutter build ios        # Build iOS (macOS only)
flutter analyze          # Run static analysis (uses flutter_lints)
flutter test             # Run tests
```

Firebase is configured via `lib/firebase_options.dart` (generated by FlutterFire CLI, project ID: `lostfoundapp-1e774`).

## Architecture

### App Structure

- **Entry point**: `lib/main.dart` тАФ initializes Firebase, requests camera permissions, defines named routes, and contains `HomePage` widget
- **Pages** (`lib/pages/`): all pages are StatefulWidgets using `setState()` for state management (no Provider/BLoC/Riverpod)

### Navigation

Named routes defined in `MaterialApp.routes`:
- `/splash` тЖТ SplashScreen (2s delay тЖТ SignIn)
- `/signin` тЖТ SignInPage тЖТ `/home` on success
- `/signup` тЖТ SignUpPage тЖТ `/signin` on success
- `/home` тЖТ HomePage (hub with Lost/Found buttons)
- `/lost` тЖТ LostPage (report lost items)
- `/found` тЖТ FoundPage (report found items)
- `/profile` тЖТ ProfilePage

Uses `Navigator.pushReplacementNamed()` for most transitions. Lost/Found/Profile pages have drawer navigation.

### Firebase Collections

| Collection | Fields | Purpose |
|---|---|---|
| `users` | uid, email, firstName, lastName, phone_number | User profiles |
| `lost_items` | itemName, description, location (GeoPoint), imageUrls[] | Lost item reports |
| `found_items` | itemName, description, location (GeoPoint), imageUrls[] | Found item reports |

### Firebase Storage

- `lost_item_images/{timestamp}.jpg` тАФ images for lost items
- `found_item_images/{timestamp}.jpg` тАФ images for found items

### Key Dependencies

- `firebase_auth`, `cloud_firestore`, `firebase_storage`, `firebase_core` тАФ Firebase services
- `google_maps_flutter` тАФ map-based location picking
- `geolocator` тАФ device GPS location
- `image_picker` тАФ camera/gallery image selection
- `permission_handler` тАФ runtime permission requests
- `flutter_map` + `latlong2` тАФ alternative map support

### Patterns

- LostPage and FoundPage are structurally identical (different Firestore collections and Storage paths)
- Images named by `DateTime.now().millisecondsSinceEpoch` for uniqueness
- Location stored as Firestore `GeoPoint` for geospatial capability
- SnackBar used for user feedback on success/error
- SDK constraint: `>=3.3.0 <4.0.0`
